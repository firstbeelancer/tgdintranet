## Что делаем

Корпоративный интранет-хаб для быстрого доступа к внутренним сервисам компании:

- лента объявлений для всех сотрудников;
- каталог быстрых ссылок на рабочие сервисы (CRM, GitHub, Figma, аналитика, 1С и др.);
- контакты сотрудников с поиском по отделам;
- шаблоны документов и заявлений с категоризацией;
- база знаний (регламенты, гайды, онбординг, политики);
- маркетинговые материалы (для роли `marketer`);
- HR-раздел (для роли `admin`);
- админ-панель для управления пользователями (только `admin`).

## Для кого

- **Сотрудники TGD Apps** — все роли.
- **Администраторы** — управление пользователями и контентом.
- **Маркетологи** — управление маркетинговыми материалами.

## Ключевые сценарии

1. **Вход и навигация**
   - Сотрудник открывает `intranet.tigerapps.pro`, попадает на экран логина.
   - Вводит корпоративный email и пароль → получает JWT.
   - Видит главную с объявлениями и быстрыми ссылками, доступными его роли.

2. **Управление пользователями (admin)**
   - Админ заходит в `/admin` → создаёт нового пользователя (email, пароль, имя, роль, должность, отдел).
   - Видит таблицу всех пользователей с фильтрацией по ролям.

3. **Публикация объявления (marketer / admin)**
   - Маркетолог или админ создаёт объявление → выбирает минимальную роль, которая его увидит.
   - Сотрудники с ролью ≥ выбранной видят его на главной.

4. **Просмотр контактов**
   - Любой авторизованный пользователь ищет коллег по имени / должности / отделу.
   - Видит email, телефон, Telegram, дату рождения.

5. **Доступ к шаблонам документов**
   - Все авторизованные видят общие шаблоны (отпуск, командировка, закупка).
   - Маркетологи дополнительно видят маркетинговые шаблоны.

## Ролевая модель

| Роль | Доступ |
|---|---|
| **user** | Главная, Сотрудники, Документы, База знаний (видимая user-уровню) |
| **marketer** | Всё что user + Маркетинг + документы для маркетинга + страницы БЗ с visible_to ≥ marketer + может создавать объявления |
| **admin** | Полный доступ, включая HR, Админ-панель, управление пользователями, удаление контента |

Иерархия ролей: `admin > marketer > user`. Реализуется через уровни:
- `user` = 1
- `marketer` = 2
- `admin` = 3

Контент с полем `visible_to = X` виден пользователям с ролью ≥ X.

## Стек

- **Frontend**: React 18 + TypeScript + Vite + TailwindCSS + Radix UI (минимум shadcn)
- **Backend**: Node.js 22 + Express + TypeScript + Drizzle ORM + PostgreSQL 16
- **Auth**: JWT (12h) + Argon2 для паролей
- **Deploy**: Docker Compose, Coolify (автодеплой из GitHub `main`)
- **Хранилище**: PostgreSQL volume (только локально, без S3 на прототипе)

## Ограничения прототипа

- Без S3 — файлы шаблонов пока хранятся как внешние ссылки.
- Без OAuth / SSO — только локальная JWT-авторизация.
- Без 2FA.
- Без email-уведомлений.
- Без интеграции с CRM / Bitrix — только ссылки.
- Без реальной загрузки аватарок.
- Без мобильного приложения (только web responsive).
- Без i18n (только русский).
